HTML


Kapitel 1 - Was ist HTML?


Mit HTML kann man das Grundgerüst einer Webseite coden. Bei HTML gibt es sogenannte Elemente.

Der öffnende Tag besteht aus einer öffnenden Spitzen Klammer, danach kommt das Zeichen und zuletzt die schliessende Spitze Klammer.

Der schliessende Tag besteht aus einer öffnenden Spitzen Klammer, gefolgt von einem Schrägstrich. Nach dem Schrägstrich kommt das Zeichen und wieder die schliessende spitze Klammer.

Zusätzlich gibt es bei einem Element auch noch einen Attributnamen und ein Attributwert.

Der Attributname zeigt, was für eine Art Zusatzinformation angegeben wird und der Attributwert ist die Information oder Einstellung des Attributs

Kapitel 2 - Text


Um bei einem Text Überschriften zumachen, gibt es folgende Elemente:

  • <h1>Überschrift 1</h1>

  • <h2>Überschrift 2</h2>

  • <h3>Überschrift 3</h3>

  • <h4>Überschrift 4</h4>

  • <h5>Überschrift 5</h5>
  • <h6>Überschrift 6</h6>

<h1></h1> ist die grösste Überschrift und wird immer kleiner.

Um einen Absatz zu erstellen, muss man den Text mit einem öffnenden <p> und einem schliessenden </p>-Tag umgeben

Man kann auch einen Text mit HTML gestalten, dies eignet sich bei einzelnen Worten.

<sub>tiefgestellt</sub>

Kapitel 3 - Listen


Bei HTML gibt es drei Arten von Listen: geordnete Listen, ungeordnete Listen und Definitionslisten.

Geordnete Listen:

Sie werden mit dem Element <ol> erstellt.
Jeder Eintrag wird zwischen dem öffnenden Tag <li> und dem schliessenden Tag </li> gestellt.
Die Aufzählungszeichen sind standardmässig nummeriert, diese Zeichen kann man aber im CSS bei list-style-type ändern.

<ol>

  1. <li>Gemüse</li>
  2. <li>Früchte</li>
  3. <li>Fleisch</li>
</ol>

Ungeordnete Listen:

Sie werden mit dem Element <ul> erstellt.
Jeder Eintrag wird zwischen dem öffnenden Tag <li> und dem schliessenden Tag </li> gestellt.
Die Aufzählungszeichen sind Punkte.

<ul>

  • <li>Gemüse</li>
  • <li>Früchte</li>
  • <li>Fleisch</li>
</ul>

Definitionslisten:

Sie werden mit dem Element <dl> erstellt.
Der Begriff, der definiert wird, steht zwischen dem öffnenden <dt> Tag und dem schliessenden </dt> Tag.
Die Definition wird von <dd> umschlossen.

<dl>

<dt>Subventionen</dt>
<dd>Zuschüsse vom Staat</dd>
</dl>

Kapitel 4 - Links


Links werden mit dem Element <a href=“www.google.ch“> Zu Google </a> erstellt. Es gibt eine klar definierte Verzeichnisstruktur. Die verschiedenen Ebenen sind dabei mit familiären Ausdrücken benannt, mit dem Überordner als Mutter, dem Unterordner als Kind etc. Diese Verzeichnisstruktur muss unbedingt beachtet werden, da diese für die Pfadangabe für z.B. ein Bild unerlässlich ist. Der Pfad muss vollständig und korrekt angegeben werden.
Es gibt die absoluten und relative URLs.
Die relativen URLs werden für Links auf der eigenen Seite gebraucht, z.B. für das Anzeigen eines Bildes oder für das Wechseln auf eine Unterseite der eigenen Seite. Ein Beispiel für einen relativen Pfad ist: ../test_ordner/style.css.
Die absoluten URLs werden gebraucht, um z.B. eine andere Webseite auf der eigenen zu verlinken. Ein Beispeil dafür ist: https://www.google.com.

<a> Link einfügen
<a href=“mailto : Mailadresse“>E-Mail an</a> : Mail an Person schicken (wird heute selten verwendet)
<a href=“www.google.ch“ target=“_blank“>Zu Google</a> : Target macht, dass der Link in einem neuen Fenster geöffnet wird.

Ausserdem kann zu bestimmten Teilen auf derselbigen Seite Seite gewechselt werden, d.h, man kann zu einem bestimmten Punkt auf der eigenen Website springen. Dies wird gemacht mit den Attributen id und href.
Um auf eine andere Website wechseln zu können, muss man das Attribut href und einen absoluten Pfad verwenden. Wenn diese andere Website in einem neuen Tab geöffnet werden soll, muss das Attribut target hinzugefügt werden.

Kapitel 5 - Bilder


Wenn man Bilder auf die Website stellen will, gibt es Dinge, die wichtig zu beachten sind:

  • Urheberrecht beachten
  • Bildformat beachten
  • Speicherort beachten

Um das Urheberrecht nicht zu verletzen, können Fotoagenturen genutzt werden oder es muss genaustens auf die Lizenierung des Bildes geachtet werden.
Bilder werden eingefügt mit dem Element <img>. Das Attribut src gibt den Speicherort des Bildes an und ist notwendig, um das Bild anzuzeigen. Mit dem Attribut alt kann ein alternativtext angezeigt werden, falls das Bild nicht angezeigt wird.
Um das Bild auch ganz und ordnungsgemäss anzuzeigen, gelten hier wieder drei wichtige Regeln:
  • Bild im richtigen Format speichern
  • Bild in der richtigen Grösse speichern
  • Die richtige Bildauflösung verwenden

Dazu könne gut Editoren genutzt werden, mit ihnen lassen sich die Bilder bearbeiten und anpassen, bis das optimale Format für die Seite erreicht wird.
Es gibt noch einen Haken: Die verschiedenen Bildformate. Je nach Format sind bestimmte Bilder für bestimmte Zwecke sinnlos oder gar nutzlos. Diese drei Formate lohnt es sich zu kennen:
  • JPG - JPG-Grafiken sind für den gebrauch von sehr farbigen Bildern wie Fotos sehr gut geeignet.
  • GIF – GIFs sind häufig animierte Bilder, die einen Bewegungsablauf haben. Sie sind z.B. nützlich, um einen kurzen Ablauf bildlich darzustellen.
  • PNG – PNG-Grafiken machen sich gut als Logos.

Jedes Bild und jeder zusätzliche Frame verlängert die Ladezeit der Seite.

<figcaption> Bildunterschrift

Kapitel 7 - Formulare


in eine Webseite einzubauen ist ein guter Weg, um Informationen vom User zu bekommen aber zieht auch einige Sachen mit sich zu beachten sind folgende Punkte:

  • Wenn man Nutzerdaten speichert ist man automatisch für deren Sicherheit verantwortlich.

  • Wenn man Nutzerdaten speichert muss man dem Benutzer mitteilen was für Daten gespeichert werden, wozu sie gebraucht werden und an wen man sie weitergibt

  • Daten sollten generell verschlüsselt übertragen werden, vor allem Sensitive.

Formulare bilden sich aus einem <form>-Element in welches man verschiedene <input>-Elemente einfügt sowie ein<button type=“submit“>-Element um die Eingabe zu bestätigen und das Formular abzuschicken. Mit einem <label>-Element kann man dann dem Benutzer anzeigen was in das Feld eingegeben werden sollte.

Mit hilfe des <fieldset>-Element kann man verschiedene <input>-Elemente Gruppieren und mit hilfe des <legend>-Elements diese Gruppe beschriften.

Es gibt verschiedene type=““ Attribute welche man verwenden kann um verschiedene eingaben zu tätigen:

Type=“text“

Es können einfache Texteingaben gemacht werden.

Type=“password“

Für Passworteingaben, der Text wird als ****** angezeigt aber trotzdem unverschlüsselt übertragen.

<textarea>

Wie type=“text“ aber grösser und ein eigenes Element.

Type=“radio“

Fügt Radio Knöpfe ein von denen nur einer ausgewählt sein kann.

Jedoch macht dies nur die Knöpfe um sie zu beschriften braucht man noch ein <label>-Element

<label>

Mit einem <label>-Element kann man <input>-Elemente beschriften mit Hilfe der id

Type=»checkbox»

Checkboxen sind wie Radio Buttons einfach dass man mehrere auswählen kann.

<select> & <option>

Mit einem <select>-Element kann man ein Dropdown Menü machen aber um darin Optionen machen zu können braucht man <option>-Elemente.

Durch das hinzufügen von multiple=“multiple“ lassen sich mehrere auswählen.

Type=“file“

Erlaubt es dem User Dateien vom System hochzuladen

Type=“submit“

Um ein Formular abzuschicken muss man auf den submit Button drücken

Type=“hidden“

Verstecktes Eingabe Feld, kann verwendet werden um lokal gecashte Daten mit dem Formular zu übermitteln

Required=“required“

Wenn dieses Attribut in einem <input>-Element oder ähnlichem ist kann das Formular nicht abgesendet werden wenn keine Eingabe / Auswahl getroffen wurde.

Type=“date“

Ein Datum kann ausgewählt werden

Type=“email“

Eingabefeld das nur E-Mail Formate anerkennt.

Type=“url“

Eingabefeld das nur URLs anerkennt.

Type=“search“

Einzeiliges Textfeld, braucht noch zusätzlichen Code für Funktionalität.

placeholder=““

Fügt einen Platzhalter Text in Formular Felder ein der Verschwindet bei einer Eingabe

CSS


Kapitel 1 - Was ist CSS?


CSS ist das, was eine Website schliesslich ästhetisch macht. CSS steht für 'Cascading Style Sheets'.
Im CSS werden Regeln angewandt. Die Regeln bestehen aus einem Selektor und einer Deklaration. Die Deklaration wird von gescheiften Klammern umfasst und besteht aus einer Eigenschaft und einem Wert.

Kapitel 2 - Farbe


In CSS können Farben auf drei Arten festegelegt werden, nämlich durch RGB-Werte, Hex-Codes und Farbennamen.

RGB-Werte:

Die Werte für den Rot-, Grün- und Blauanteil werden mit Zahlen zwischen 0 und 255 angegeben.

rgb (102,205,170)

Hex-Codes:

Hier werden die Werte für den Rot-, Grün- und Blauanteil in Hexadezimal angegeben.

rgb #66cdaa

Farbnamen:

Die Farben werden durch vordefinierte Namen dargestellt.

MediumAquaMarine

Deckkraft:

Mit RGBA kann man die Deckkraft der Farbe bestimmen.

rgba (102,205,170,0.5) --> Hier ist die Deckkraft jetzt 50% und die Farbe MediumAquaMarine.
Die ertsen drei Ziffern sind RGB und die letzt steht für die Deckkraft.

Kapitel 6 - Layout


CSS behandelt alle Elemente so, als befänden sie sich jeweils in ihrem eigenen Kasten. Dabei wird zwischen den Block- und Inlineelementen unterschiedet. Blockelemente beginnen immer in einer neuen Zeile. Dazu gehören z.B.

,

,

    ,
  • . Inline-Elemente ‘schwimmen’ zwischen den umgebenden Elementen; Das sind z.B. , , .

    Ausserdem ist von ‘umfassenden Elementen’ die Rede. Wenn sich ein Blockelement im Anderen befindet, wird der äussere Kasten als das Umfassende-, Einschließende- oder Elternelement bezeichnet.

    Bei der Positionierung spielen mehrere Verfahren eine wichtige Rolle; der normale Fluss, die relative, absolute und feste Position und das schwimmende Element.

    Der normale Fluss ist standardmässig aktiviert. Jedes Blockelement kommt auf eine neue Zeile, selbst wenn zwei Blockelemente noch Platz nebeneinander haben.

    Die relative Positionierung ist gut, um Elemente beliebig zu versetzen. Die Positionen der Umgebenden Elementen bleiben dabei unverändert.

    Eine Absolute Positionierung wird gebraucht, wenn das Element immer dableiben soll, wo es ist. Dabei werden die umfassenden Elemente nicht beeinflusst.

    Feste Positionierungen sind den absoluten sehr ähnlich, mit dem Unterschied, dass sie beim Scrollen mitkommen.

    Die schwimmenden Elemente werden aus dem normalen Fluss herausgenommen und können von den umgebenden Elementen umflossen werden. Die schwimmenden Elemente werden entweder ganz links oder ganz rechts platziert.

    Position:static – normaler Fluss
    Position:relative – relative Positionierung
    Position:absolute – absolute Positionierung
    Position:fixed – feste Positionierung
    z-index – überlappende Elemente; Bei relativen, festen und absoluten Positionen können sich Elemente überlappen. Um nun selbst zu definieren, welches Element zuoberst liegt, wird nun der z-index verwendet. Je höher der Wert des z-index, umso höher die Ebene des Elements.
    flaot – schwimmende Elemente
    clear – schwimmende Elemente freistellen:
    • left – linke Kante darf nichts berühren
    • right – rechte Kante darf nichts berühren
    • both – linke und rechte Kante darf nichts berühren
    • none – die Elemente dürfen einander berühren

    Stylesheets: Mit @import können andere Regeln importiert werden. Mit link werden die einzelnen Stylesheets mit dem HTML-Code verlinkt.
